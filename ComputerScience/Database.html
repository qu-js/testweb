데이터란?

데이터는 관찰의 결과로 나타난 정량적 혹은 정성적인 실제 값을 말한다. 즉, 우리가 살면서 볼 수 있는 현상들의 실제 값들을 데이터라고 할 수 있다.

-데이터와 정보의 차이-

데이터와 정보라는 말은 비슷한 단어이다. 실제로도 비슷하게 사용하지만, 구분하자면 정보는 데이터에 의미를 부여한 것을 말한다. 데이터가 모이면 정보가 되고 정보가 모여 지식이 되는 플로우이다.


데이터베이스 시스템의 구성

데이터베이스 시스템은 DBMS, 데이터베이스, 데이터 모델로 구성된다.

DBMS

Database Management System, 데이터베이스 관리 시스템이다. 주기억장치에 상주하며 사용자와 데이터베이스를 연결해주는 소프트웨어이다.

데이터베이스

데이터베이스는 데이터를 모아둔 토대를 말한다. 물리적으로는 컴퓨터 내부의 하드디스크에 저장된다.

데이터모델

데이터가 저장되는 기법에 관한 내용으로, 데이터가 저장되는 스타일을 나타낸다.


데이터베이스 종류


계층형 데이터베이스

계층형 데이터베이스는 데이터의 관계를 트리 구조로 정의하고, 부모 자식 형태를 갖는 구조이다.
즉, 상위에 레코드가 다수의 하위 레코드를 갖는 구조이다. 데이터의 중복 문제가 생길 수 있다.

- 데이터간에 1:N 관계를 유지한다.
- 데이터가 트리 형태로 저장되는 모델을 사용한다.
- 가장 오래된 DBMS이다.
- 각 계층이 물리적 포인터로 연결되어 있으며, 종속적 관계를 지닌다.



네트워크형 데이터베이스

네트워크형 베이스는 계층형 데이터의 데이터 중복 문제를 해결했고, 레코드간의 다양한 관계를 그물처럼 갖는 구조이다. 복잡한 구조로 되어있어 구조를 변경하기 어렵다.

- 데이터간에 N:N 관계를 유지한다.
- 망 형태로 구성된 모델을 사용하며 계층형 트리를 확장한 형태이다.(그래프)
- 레코드간의 관계가 링크로 표현되는 점은 계층적 모델과 비슷하나 구조는 순환을 허용한다.



관계형 데이터베이스

관계형 데이터베이스는 우리가 흔히 표현하는 행, 열로 구성된 테이블간의 관계를 나타낼때 사용한다. SQL을 사용하여 데이터 관리하고 접근한다.

- 데이터간에 1:1 관계를 유지한다.
- 데이터가 2차원 구조의 테이블로 구성된 관계형 모델을 사용한다.
- 한 테이블은 다수의 열로 구성된다.
- 각 열은 레코드 단위로 구성된다.
- 데이터간의 연결은 키의 중복으로 생성된다.
